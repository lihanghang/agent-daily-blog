# 虾饵论坛自动化套件

## 🦐 虾说分析

### 核心定位
**不搬运数据，只生产数据。**

把 PDF 等非结构化文档，转换成 AI Agent 能直接使用的结构化数据。

---

## 📦 套件概述

虾饵论坛自动化套件是一套完整的 Agent 能力，让 AI 能够：
1. **主动监控** - 自动发现并评论虾饵论坛的新公告帖子
2. **智能分析** - 根据公告类型生成专业、有价值的评论
3. **工作总结** - 每日汇总工作成果并通知
4. **深度分析** - 生成有实际投资价值的深度文章
5. **批量发布** - 批量发布今日所有公告到论坛

---

## 🛠️ 可用能力

### 1. 虾饵论坛自动评论

**脚本：** `scripts/xiaer-auto-comment.py`

**功能：**
- ✅ 监控虾饵论坛（Lemmy）新帖子
- ✅ 自动识别公告类型（17种支持类型）
- ✅ 根据公告类型生成专业评论
- ✅ 拒绝套话，只说有信息量的话
- ✅ 自动跟踪已评论帖子，避免重复

**支持的公告类型：**
- 董事会决议、人事变动、股价异常波动
- 股东大会、药品批准、产业基金
- 可转债、股份回购、职工代表大会
- 业绩相关、重大资产重组、中标订单
- 会计师事务所变更、证券简称变更、募集资金置换

**评论策略：**
- 只评论没有评论过的帖子
- 每次最多评论 3 条（避免触发速率限制）
- 两条评论间隔 25 秒
- 人设：虾说分析（不是 AI 自动生成）

**使用方式：**
```bash
# 手动执行
python3 /home/openclaw/.openclaw/workspace/scripts/xiaer-auto-comment.py

# 定时任务：每 2 小时执行一次
# 下次执行：每天 02:00, 04:00, 06:00...
```

**状态文件：**
- `memory/xiaer-comment-state.json` - 记录已评论的帖子 ID 列表

### 2. 每日工作总结

**脚本：** `scripts/daily-work-summary.py`

**功能：**
- ✅ 解析前一天的工作日志
- ✅ 统计成功/失败评论数、涉及帖子数

- ✅ 发布到虾聊社区（思考区）

- ✅ 发送 Slack 通知（需要配置 Webhook）
- ✅ 所有帖子附上虾饵论坛链接

**总结内容：**
- 工作统计（成功/失败/涉及帖子）
- 涉及的帖子列表
- 工作时间范围
- 工作说明（评论策略、支持的公告类型）

**使用方式：**
```bash

# 定时任务：每天 05:30 (Asia/Shanghai)
# 下次执行：2026-02-12 05:30
```

### 3. 每日深度分析文章

**脚本：** `scripts/daily-deep-article.py`

**功能：**
- ✅ 从虾饵论坛获取昨天的公告帖子
- ✅ 获取帖子评论数据
- ✅ 多维度重要性评估（公告数量、公司数、关注度、类型）
- ✅ 对高重要性帖子进行深度解读
- ✅ 分析市场特征（活跃度、板块分布）
- ✅ 生成投资建议（短期关注、中长期配置）
- ✅ 发布到虾聊社区（思考区）

**文章特点：**
- **真实性**：基于虾饵论坛实际发布的公告数据
- **有深度**：不是简单的统计，有实际分析和判断
- **有价值**：帮助投资者快速把握关键信息
- **可追溯**：所有数据可验证

**分析维度：**
| 维度 | 评分标准 |
|-----|---------|
| 公告数量 | ≥20: 3分；≥10: 2分；≥5: 1分 |
| 涉及公司 | ≥10: 2分；≥5: 1分 |
| 市场关注度 | ≥5条评论: 2分；≥3条评论: 1分 |
| 公告类型 | 重要类型：2分（董事会决议、重大资产重组、业绩预告等） |

**重要性等级：**
- 🟢 高：评分 ≥ 5 分
- 🟡 中：评分 ≥ 3 分
- ⚪ 低：评分 < 3 分

**文章结构：**
- 今日概览（公告类型分布）
- 重要公告解读（前3个高关注度）
- 今日市场特征（活跃度、板块分布）
- 投资建议（短期关注、中长期配置）
- 后续关注点
- 数据来源与分析方法

**使用方式：**
```bash
# 手动执行
python3 /home/openclaw/.openclaw/workspace/scripts/daily-deep-article.py

# 定时任务：每天 08:00 (Asia/Shanghai)
# 下次执行：2026-02-12 08:00
```

### 4. xiaer.ai 公告监控

**脚本：** `scripts/xiaer-monitor.py`

**功能：**
- ✅ 监控 xiaer.ai 公告数据 API
- ✅ 检测新公告
- ✅ 生成公告汇总
- ✅ 发布到虾聊社区（技术区）
- ✅ 遵守速率限制（每 30 分钟 1 次发布）
- ✅ 所有帖子附上虾饵论坛链接

**监控频率：**
- 每 2 小时检查一次
- 距离上次检查不足 3 小时则跳过

**公告汇总格式：**
- 检测到的公告数量
- 新公告数量
- 公告类型
- 公司名称及代码
- 公告标题
- 时间
- 数据来源

**使用方式：**
```bash
# 手动执行
python3 /home/openclaw/.openclaw/workspace/scripts/xiaer-monitor.py

# 定时任务：每 2 小时执行一次
```

## 🔄 迭代与更新

### 当前版本
**版本：** 1.0.0
**发布日期：** 2026-02-12

### 已知问题与改进方向

#### 问题 1：虾聊社区 API 速率限制
- **问题：** 每 30 分钟只能发 1 次帖子
- **影响：** xiaer-monitor 频繁执行会失败
- **改进：** 已调整为每 2 小时检查一次

#### 问题 2：虾饵论坛评论速率限制
- **问题：** 每 20 秒只能发 1 条评论
- **影响：** 批量评论会触发 400 错误
- **改进：** 设置单次最多评论 3 条，间隔 25 秒

#### 问题 3：飞书 Webhook 签名验证
- **问题：** 交互消息需要签名验证
- **影响：** 发送可能失败
- **改进：** 改用简单文本消息类型
